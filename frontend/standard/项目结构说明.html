# 项目简介

  • 本项目用Vue.js官方提供的命令行工具vue-cli结合前端打包工具webpack构建，引入vue-router、vuex插件管理路由和数据流，使用饿了吗开源的element作为UI组件库。
  • 首次运行先安装node环境，再运行npm install安装相关node模块。
  • 开发时运行:npm run dev或npm start
  • 打包命令:npm run build
  • 服务器部署:将打包好的dist文件夹放入服务器对应文件夹即可运行。

# 文件结构说明

# 一、build：webpack打包相关配置

  • build.js：运行npm run build时执行的文件，用于打包整个项目生成dist文件夹。
  • check-versions.js: 用于检测node、npm、webpack及所有依赖包的版本信息，若版本过低或其他依赖包需要更高版本的时候会提示升级，项目运行或打包时webpack会执行调用。
  • dev-client.js: 用于开发时的热更新（代码变更时浏览器自动刷新），项目运行或打包时webpack会执行调用。
  • dev-server.js: 利用express（一个node框架）启动一个本地服务供开发调试使用，运行npm run dev时调用此文件。
  • utils.js: webpack要用到的公用配置文件，包含了一些开发和打包时的一些公用方法。
  • vue-loader.conf.js: webpack加载.vue文件的配置（开发和生产模式不同）。
  • webpack.base.conf.js: webpack的基础配置（包含项目入口js文件、是否打包某些库文件、输出文件路径和名称配置、通用路径配置、文件加载器配置）。
  • webpack.dev.conf.js: webpack开发环境相关配置。
  • webpack.prod.conf.js: webpack生产环境相关配置（即项目打包时的配置，因为用于服务器部署涉及到一些优化，在此可配置如是否开启source-map，是否删除空格注释以 及文件压缩等特殊配置）。

# 二、config：webpack配置文件

  • dev.env.js:配置说明为开发环境。
  • index.js:webpack配置入口文件，包含了最基础的配置（如指定静态文件夹名为static，开发环境监听端口及是否启用热更新等）。
  • prod.env.js:配置说明为生产环境。

# 三、dist：webapck打包后生成的文件夹，用于服务器部署。

  • index.html: 部署到服务器的访问入口文件。
  • static: 静态资源文件夹
  • css：打包后的样式文件（已压缩）
  • element-theme：饿了吗的主题样式文件，用于切换皮肤后饿了吗组件做相应切换。
  • fonts:公用的字体文件。
  • img:打包后，项目中实际用到的图片（图片已被重命名）。
  • js:打包后的js文件。 --vendor.js:公用库文件（echarts、jquery、lodash等）打包后生成的文件。 --manifest:打包后的js文件依赖关系说明，方便js文件在运行过程中更快速的查找依赖关系，提供运行效率。 --app.js:项目打包后首次运行需要用到的必要配置。 其他:在路由文件配置了按需加载，其他js文件对应具体的业务模块，当运行到某个模块时才去加载对应的js文件。 --Print-template:打印模板相关文件。 --favicon.ico:项目运行时浏览器标签左上角的小图标。 --global.js:全局配置（用于项目打包后更改api接口的访问地址）

# 四、node_modules:开发相关依赖包（此文件夹只在开发时存在，npm自身原因文件夹目录结构嵌套比较深，不上传到svn和直接copy）

# 五、src:项目开发文件

  • api:接口相关文件 api.js:接口分发中心，将各个模块的接口集中到此文件，方便查找和开发，项目中用到的所有接口方法都可以在此文件找到。 config.js:接口相关配置，包含后端接口ip地址的配置（可在global.js中修改ip地址来覆盖此处的默认配置），axios实例相关配置，包含请求配置token、请求拦截、响应数据拦截等。 consult.js:远程医疗模块相关接口。 examine.js:审核端模块相关接口。 login.js:登录相关接口。 medical.js:病历收藏模块相关接口。 patientcenter.js:患者中心模块相关接口。

  • assets:资源文件 css:公用的样式文件，目前只包含reset.css，用于重置、统一一些浏览器默认样式，以及一些约定成俗的样式重置。 iconfont:项目中用到的字体文件及其demo，打包后只留下字体文件，对应打包后dist中的fonts文件夹。 images:图片文件夹，开发时页面用到的背景图和一些图标图片都在此文件夹，打包后用到的图片会被重命名放入dist的img文件夹。 skin:自定义换肤功能的样式文件。 --color.scss:定义了四种不同皮肤的配色变量。 --skin.scss:将换肤相关的页面组件（结构）样式放到此文件统一管理。

  • common:全局公用方法和一些工具类函数 bus.js:全局巴士，实际上是新建了一个Vue实例命名为bus，用于简单的组件间通信。 directive.js:自定义指令文件，Vue允许用户自定义一些具有特殊功能（性质）的页面元素，比如自动获取焦点的input输入框。 filter.js:自定义过滤器，比如通过使用过滤器你可以在页面直接把后端返回的'1'和'2'转换成'男'和'女'或者把日期时间转换成'今天'、'明天'、'后天',而不用在每个地方都写一次转换代码。 util.js:工具类函数，比如对一些数组的特殊处理、对象特殊格式化处理、正则匹配等公用方法。

  • components:全局公用组件 404.vue:404页面。 Logo:logo组件，比如要修改logo图片或样式的时候，只用修改此文件，而不用在每个用到的地方都去修改。

  • mock:一个通过模拟接口数据的组件（插件），前端在开发过程中可以通过模拟一些接口数据来调试页面展示效果。

  • views:所有业务模块的Vue单文件组件（在Vue中所有的页面都被定义为一个单独的组件可以相互调用，而又相互独立，达到解耦的目的）。 centralit:患者中心模块所有页面（组件） consult:远程会诊模块所有页面（组件） medical:病案收藏模块所有页面（组件） operation:审核端模块所有页面（组件） referral:双向转诊模块所有页面（组件） Home.vue:项目首页 Login.vue:登录页面 Main.vue:业务模块页面容器（业务模块通过嵌套路由分发各自的菜单栏、导航栏和内容栏） Navtop.vue:公用导航栏组件 Passworld.vue:密码修改页面

  • vuex:全局（数据）状态管理插件，此插件由Vue的作者尤雨溪自己开发；用于复杂项目的数据流状态管理，简单的组件间通信用全局bus即可。 action.js:操作行为处理模块。负责处理Vue Components接收到的所有交互行为。包含同步/异步操作，支持多个同名方法，按照注册的顺序依次触发。向后台API请求的操作就在这个模块中进行，包括触发其他action以及提交mutation的操作。该模块提供了Promise的封装，以支持action的链式触发。 getters.js:state对象读取方法。图中没有单独列出该模块，应该被包含在了render中，Vue Components通过该方法读取全局state对象。 store.js:通过vuex构建的实例对象。

  • APP.vue:整个项目的根组件，所有的业务模块组件和其他功能组件都通过路由视图的方式包含在此组件中，全局公用样式和字体文件也在此文件引入。

  • main.js: 项目起始文件，引入vue和其相关插件、路由文件以及第三方公用库和用户自定义指令、过滤器等；构建Vue实例并挂在到index.html，这就是一个SPA(单页应用程序)了。

  • routes.js:路由配置文件，将各个业务模块的单文件组件通过异步方式引入（也可同步，但打包后js文件较大，首次加载速度很慢），并根据vue-router的配置规则配置好各个页面及其子页面的嵌套以及对应关系，供vue调用分发视图、路由拦截，以及axios做数据拦截时的路由跳转和其他需要用到路由相关的地方。

# 六、standard: 项目开发规范和其他一些说明文件

# 七、static:静态资源文件夹（开发时），此文件夹在webpack中配置指定，各个组件中可以直接调用此文件夹中的资源文件，webpack不对其内容做特殊处理，适合放一些不需要编译打包的文件，打包后其内容会直接放入dist的static文件夹中。

# 八、index.html:项目入口文件（开发时），与打包后dist文件夹的index.html相对应，webpack会将打包编译的css及js文件自动注入到此页面。

# 九、package.json: 项目初始化文件，包含项目基本信息说明（项目名称、版本号等）、开发依赖包及项目依赖包；开发及打包相关node命令配置说明。
